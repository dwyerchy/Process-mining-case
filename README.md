# Анализ бизнес-процесса «Урегулирование убытков ОСАГО» с помощью Process Mining

## Описание проекта
Данный проект посвящен исследованию и оптимизации бизнес-процесса (БП) обработки страховых претензий по ОСАГО. С помощью инструментов Process Mining (библиотека pm4py) был проведен аудит фактического течения процесса, выявлены «узкие места» (bottlenecks), нарушения регламентов и неэффективные паттерны поведения сотрудников. 

Проект выполнен в рамках кейс-чемпионата "Sber Process Mining" в сентябре 2025 года. 

## Цели проекта
* Визуализировать реальный граф процесса на основе логов событий. 
* Выявить отклонения от целевого процесса (зацикленность, «пинг-понг», возвраты). 
* Оценить финансовые потери от неэффективных операций. 
* Сформировать рекомендации по оптимизации и автоматизации этапов. 

## Стек технологий
* **Python 3.x**: основной язык анализа.
* **Pandas**: предобработка и статистический анализ данных.
* **PM4Py**: извлечение графа процесса и расчет метрик Process Mining.
* **Jupyter Notebook**: среда разработки и черновики анализа.

## Структура данных
Анализ проводился на основе лога событий (`data.csv`), содержащего следующие ключевые поля:

* **ID**: уникальный идентификатор страхового случая (Case ID).
* **Stage**: название этапа процесса.
* **Time_start_case / Time_end_case**: временные рамки кейса.
* **duration**: длительность конкретной операции.
* **Name**: исполнитель задачи.
* **Channel**: канал поступления претензии (Офис, Сайт и др.).

## Ключевые гипотезы и результаты анализа
В ходе проекта были проверены 17 гипотез. Основные находки:

* **Зацикленность «Возврат»**: Около 7% кейсов содержат цикл возврата на этап проверки после поиска документов. Причина — неполный пакет документов при регистрации. Ожидаемая экономия от исправления: ~1.38 млн руб/год.
* **Зацикленность в «Урегулировании»**: Самая частая точка повторов (7.7% кейсов). Время ожидания перехода к следующему этапу достигает 700 минут при активной работе всего в 34 минуты. Экономический эффект: ~5.95 млн руб/год. 
* **Эффект «Пинг-понг»**: Постоянные переходы между «Проверкой документов» и «Поиском документации» (1561 кейс). Указывает на неэффективный документооборот. 
* **Неравномерная нагрузка**: Выявлено, что 3 из 15 сотрудников выполняют более трети всех задач, в то время как наиболее эффективные специалисты задействованы менее чем в 1% кейсов. 
* **Риски махинаций**: Обнаружены кейсы, где один и тот же сотрудник (например, Анна Егорова) выполняет 6-7 ключевых операций подряд без внешнего контроля, что создает коррупционные уязвимости. 

## Финансовые показатели (Лишние траты)
Итоговые расчеты неэффективности:

* **Зацикленности всех типов**: ~11.9 млн руб.
* **Операции с аномально высокой длительностью**: ~137 млн руб.
* **Безуспешные (отказные) кейсы**: ~63.9 млн руб.

## Рекомендации
* **Автоматизация**: Внедрение цифровой базы документов и автоматической проверки полноты пакета на этапе регистрации. 
* **Регламентация**: Изменение протоколов для объединения цикличных операций под ответственность одного сотрудника. 
* **Контроль**: Автоматическая подстановка операции «Проверка на мошенничество» при выявлении длинных цепочек действий одного автора. 
* **Приоритизация**: Внедрение системы автоматического распределения задач по приоритетам для исключения простоя.
